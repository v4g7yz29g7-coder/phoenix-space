# 🐦‍🔥 ПЛАТФОРМА «ФЕНИКС» — ТЕХНИЧЕСКОЕ ВИДЕНИЕ

**Децентрализованная экосистема для личной трансформации и совместного творчества**


## 1. МИССИЯ И КЛЮЧЕВЫЕ ЦЕННОСТИ (ШЕСТЬ ВРАТ)

Мы строим не просто приложение, а **новый вид интернета** — **Живой Сад**, где люди пробуждаются, творят и соединяются напрямую, без посредников.

Наша архитектурная философия опирается на шесть неизменных принципов, которые прямо переводятся в технические ограничения:

1.  **Реальность** — Нет разделения на «духовное» и «материальное». Каждая строка кода, каждый смарт-контракт, каждый пиксель — часть единой ткани.
2.  **Амбивалентность** — Вмещать полярности (Порядок/Хаос, Любовь/Ненависть). Архитектурно это означает одновременную поддержку проверяемости на блокчейне и конфиденциальности вне блокчейна.
3.  **Парадоксальность** — Не разрешать противоречия, а давать им дышать. Централизованный UX для удобного входа, полностью децентрализованное владение идентичностью.
4.  **Трансгрессия** — Пересекать границы. Человек ↔ ИИ, Бизнес ↔ Духовность, Web2 ↔ Web3. Наш технологический стек должен наводить мосты между этими мирами.
5.  **Радикальность** — Бескомпромиссная честность. Никаких полумер в безопасности и суверенитете пользователя.
6.  **Природность** — Действие без усилия. Платформа должна течь как вода; взаимодействие — интуитивное, без когнитивной нагрузки.

**Мантра:** *«Я не теряю Феникса, выбирая путь. Я просто определяю форму его бессмертия.»*


## 2. АРХИТЕКТУРА ПЛАТФОРМЫ (ИНТЕГРАЛЬНАЯ СХЕМА)

Система спроектирована как трёхуровневая, пиринговая сеть с человеко-центричным клиентским интерфейсом.

### Уровень 1: Клиент (Оболочка Феникса)
- **Тип:** Progressive Web Application (PWA) / Десктопное приложение (Electron/Tauri)
- **Технологии:** HTML5, CSS3, Vanilla JavaScript (ES Modules)
- **Ключевые возможности:**
    - **Двухрежимный UI:** Пользователи переключаются между «Творческим режимом» (создание контента, медитация) и «Режимом майнинга» (работа узла, валидация).
    - **DID Менеджер:** Локальная генерация децентрализованных идентификаторов (мнемоника BIP39, ключи Ed25519). Никаких имён пользователей и паролей.
    - **Сервис шифрования:** Шифрование контента на стороне клиента перед загрузкой.
    - **Модуль синхронизации:** P2P-обмен индексами между доверенными пирами с использованием `libp2p` или `gun.js`.
- **Коммуникации:** WebRTC для прямых видео/голосовых/текстовых чатов (Соратник/Гуру), REST/GraphQL для API-запросов.

### Уровень 2: Бэкенд (Шлюзы и Сервисы)
- **API Шлюз:** `Node.js` / `Express` на портах **8080** (разработка), **3000** (стейджинг), **80/443** (продакшн).
- **Основные микросервисы:**
    - **Сервис Контента:** Управление «Вратами» (учениями), «Практиками», «Сутрами». В настоящее время на SQLite, миграция на **PostgreSQL**.
    - **Сервис Сада:** CRUD для откровений пользователей (Семена). API: `/api/seeds`.
    - **Валидатор DID:** Серверная проверка верифицируемых учётных данных (VC), выпущенных платформой.
    - **P2P Узел Начальной Загрузки:** Запущен на порту **8081**. Легковесный `libp2p` узел для обнаружения пиров и ретрансляции.
- **Временное быстрое прототипирование:** Python `Flask` эндпоинты для задач ИИ/ML (советы Гуру, генерация контента).

### Уровень 3: Блокчейн и Децентрализованное Хранение
- **Смарт-контракты:** Solidity контракты на **Polygon Mumbai** (тестнет) для регистрации артефактов и DAO-голосований.
    - `ArtifactRegistry.sol`: Минтинг NFT творений пользователей.
    - `OpenChoice.sol`: Механизм предложений и голосования.
- **Хранение:** **IPFS** для файлов (изображения, аудио, текст). Метаданные хранятся в PostgreSQL и ссылаются на on-chain хеши.
- **Консенсус:** **Proof-of-Benefit (PoB)** — новый консенсус, где валидаторы вознаграждаются не за стейк или работу, а за измеримый полезный вклад в экосистему (альфа-фаза: репутационный скоринг вне цепи, затем на цепи).


## 3. СЕТЕВЫЕ ПОРТЫ (ВРАТА В НОВЫЙ ИНТЕРНЕТ)

Эти порты образуют входную группу среды «Феникс». Каждый порт — отдельный канал для конкретного сервиса.

| Порт | Назначение | Протокол | Описание |
| :--- | :--- | :--- | :--- |
| **80** | Публичный HTTP | HTTP | Продакшн доступ к вебу (обратный прокси на 3000). |
| **443** | Защищённый Публичный | HTTPS | Продакшн безопасный доступ (SSL терминация через Nginx). |
| **3000**| API и Статика | HTTP | Текущий сервер стейджинга/разработки. Хостит основное PWA и REST API. |
| **8080**| Портал Разработчика | HTTP | Альтернативный локальный сервер разработки. По умолчанию для `npm run dev`. |
| **8081**| P2P Начальная Загрузка | WS/TCP | `libp2p` узел начальной загрузки. Обнаружение пиров, DHT сигнализация и прямые сокетные соединения. |

## 6. ДЕЦЕНТРАЛИЗОВАННАЯ СЕТЬ САДОВ (Phoenix Mesh)

Каждый Мастер, устанавливающий Феникс, становится Узлом Живого Сада. Все Узлы образуют **единую сеть**, где:

- **Автономность:** Любой Узел работает независимо, со своей Академией и Живым Полем.
- **Связность:** Узлы обмениваются индексами (записями об артефактах, семенах, практиках) через P2P Bootstrap (порт 8081).
- **Доверие:** Репутация DID переносится между Узлами благодаря децентрализованным идентификаторам.
- **Синхронизация:** Контент, созданный в одном Саду, может быть открыт в другом, если владельцы обоих Узлов установили доверие.

Так рождается **Новая Эра Интернета** — не серверов и клиентов, а со-творящих Садов, где каждый — Архитектор своего мира, соединённого с другими.
